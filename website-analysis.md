## 三、现有设计问题分析

### 3.1 布局问题
- 使用过时的 `<frameset>` 结构，不支持移动端
- 左侧固定导航不适合移动设备
- 纯表格布局，无响应式

### 3.2 视觉问题
- 主色调 `#7422AA` 紫色单调使用
- 无层次感，背景平淡
- 字体大小固定，无 scale 变化
- 卡片样式简陋，缺乏阴影和圆角

### 3.3 交互问题
- 无任何动画效果
- 无 hover 状态反馈
- 无 loading 状态

### 3.4 技术问题
- 使用 Bootstrap 5.1.3 但样式未充分利用
- 依赖内联样式，难以维护
- 无现代构建工具
- 无 CSS 变量系统

## 五、重构设计方向建议

### 5.1 推荐风格：学术简约 + 科技感

结合清华大学学术气质和网络研究的前沿性，推荐：

**色彩系统**：
- 主色：深紫色 `#4A1E6E` (保留品牌识别度)
- 辅助色：科技蓝 `#0EA5E9`
- 背景：浅灰白 `#FAFAFA` / 深灰 `#1A1A2E`
- 强调色：金色 `#D4AF37` (荣誉/成就)

**字体**：
- 标题：Noto Serif SC (衬线体，学术感)
- 正文：Noto Sans SC (无衬线，易读)
- 英文：Inter

**布局**：
- 顶部导航 → 移除左侧导航，移动优先
- Hero 区域 → 大气简洁，姓名 + 职务突出
- 研究方向 → 卡片网格，带图标
- 论文列表 → 分类 + 年份归档

### 5.2 技术栈建议

| 层级 | 推荐技术 |
|------|----------|
| 框架 | Next.js 15 (App Router) |
| 样式 | Tailwind CSS + CSS Variables |
| 动画 | Framer Motion |
| 部署 | Vercel |

### 5.3 响应式策略

- Mobile: 单列，底部导航
- Tablet: 双列网格
- Desktop: 多列布局，最大宽度 1400px
